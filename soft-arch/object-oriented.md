# ソフトウェア設計基礎 - オブジェクト指向

## オブジェクト指向とは

名前の通り，プログラムのあらゆる処理ををオブジェクトとして捉えるもの．
オブジェクトとは*データと処理をセットにしたモノ*と考えてよい．

従来(手続き型プログラミング):

料理のレシピのように，処理を順番に実行することに主眼を置いている．
例えば，「Aのデータを取ってくる」→「Bの計算をする」→「Cの画面に出力する」というもの．
すなわちデータと処理がバラバラに存在する．

デメリットとして，プログラムが大規模になると，どこでどのデータが書き換えられたか追跡しにくくなる．
このため一部を修正すると全く関係ない場所でバグが出ることが増える．
（ちなみにこの現象をスパゲッティコードと呼ぶ）

## 抽象と具体を活用した継承

Point: 
is-aの原則が成立しない場合は，継承は使うべきではない．
つまり() is a ().の文章が成立しない場合はダメ．

良い例：Car - Vehicle. Cat - Pet.
悪い例：Engine is a Car. Cat - tail.
