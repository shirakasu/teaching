# Git

## 1. バージョン管理とは？

ファイルの状態を記録し，過去の任意の時点の状態を復元できるようにする仕組み．

### Gitを使わないバージョン管理

手動でバックアップを取ると，ファイル名が煩雑になり，どれが最新か分からなくなります．
また，管理するべきファイル数も多くなってしまいます．

- `Report_20240612`
- `Report_20240614_修正`
- `Report_20240618_最終`
- `Report_20240621_本当に最終`
- `Report_20240623_これで最後....`

### Gitを使ったバージョン管理

**Gitを使うメリット:** ファイル名は1つのまま，裏側で全ての変更履歴を管理できます．

---

## 2. Gitのシステム概観と重要用語

Gitは「3つの場所」でファイルを管理します．この流れを理解すると，Gitのhelp documentも理解できると思います．

1. **ワークツリー (Working Tree):** 実際に作業しているディレクトリ．ファイルの作成や編集を行う場所．
2. **ステージングエリア (Staging Area / Index):** リポジトリに記録する前の「準備場所」．変更したファイルの中から，次の保存に含めたいものだけを選択して載せます．
3. **リポジトリ (Repository):** 変更履歴とファイルが永久に保存される場所．

注意点として，Gitはあくまでローカルのファイル・変更履歴を管理するツールです．
インターネット上でファイルを複数人で管理する際にはリポジトリ共有サービス(Github, GitLab, Giteeなど)を利用します．

## 3. Git必須コマンド 7選

これだけ覚えれば，個人の開発やチーム開発の基礎がカバーできます．

| コマンド | 役割 | 実行タイミング |
| :--- | :--- | :--- |
| `git init` | Git管理を開始する | プロジェクトの最初の一回 |
| `git add <ファイル名>` | ステージングに載せる | 保存したい変更ができた時 |
| `git commit -m "コミットメッセージ"` | リポジトリに記録する | 一区切りの作業が完了した時 |
| `git status` | 現在の状態を確認する | 「あれ、いま何したっけ？」と思った時 |
| `git log` | 過去の履歴を見る | 過去の変更点を確認したい時 |
| `git push origin <ブランチ名>` | リモートに送信する | 自分の作業をチームに共有する時 |
| `git pull origin <ブランチ名>` | リモートから取得する | 他の人の作業を自分のPCに取り込む時 |

---

## 4. 基本のルーティン（作業の流れ）

1. ファイルを編集する（ワークツリー）
2. `git add .` で変更をステージングエリアに載せる
3. `git commit -m "add: 〇〇機能を実装"` で履歴を刻む
4. `git push origin <ブランチ名>` でサーバーに送る

---

## まとめ

Gitは「**add（選ぶ）**」→「**commit（刻む）**」→「**push（送る）**」の3ステップが基本です．
もし操作を間違えても，リポジトリに保存（commit）さえしてあれば，いつでも元の状態に戻すことができるのが最大の強みです．
もし操作を間違えた場合に戻すコマンドもよく使うので，ぜひ調べてみてください．
