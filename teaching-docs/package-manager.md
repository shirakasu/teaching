# パッケージマネージャーとは

バックエンド，フロントエンド問わず，どのプログラミング言語で開発するにしても基本的には外部で公開されている様々なパッケージを利用することとなる．
すでに世の中にあるものをわざわざ0から開発する必要はない！（車輪の再開発）
ただ複数人で開発を行う際に，各々自由にパッケージをインストールするとバージョンがバラバラになってしまう．
このため，パッケージマネージャーと呼ばれるパッケージの管理，インストール，アップグレードを担ってくれるツールを使用する．

パッケージマネージャーの代表例としては以下のようなものがあります．

- npm(JavaScript製)
- gem(Ruby製)
- composer(PHP製)
などなど

ここではnpm/yarnを紹介します．
npm/yarnでは以下のようなメリットがあり，これまでの課題を解決している．

- 依存関係を意識しなくても勝手に解決してくれる．
- チーム内でのパッケージの共有や，バージョン統一が容易．
- 世界中で公開されているパッケージをコマンド1つで利用可能．
- どこから読み込んだものか分かりやすくなった．

以下のようなコマンドを使用することでパッケージのインストールができる．

```shell
npm install [パッケージ名]
```

上記のコマンド実行時に，ローカルのpackage.jsonが更新され，パッケージの情報が追記される．
同時にpackage-lock.jsonファイルが生成され，パッケージが内部で使用している別パッケージ等のバージョン情報や依存関係が追記される．

以下のようなコマンドを使用することでpackage.jsonとpackage-lock.jsonの2ファイルから複数のモジュールを展開できる．

```shell
npm install
```

コマンドを実行すると，パッケージの実態が格納されるnode_modulesというフォルダが生成される．
このフォルダはサイズが膨大になるので，通常はGithubなどの管理ツール上にあげたり，コピペはしない．
